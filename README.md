This repository contains a professional data analysis and statistical modeling project, initially developed as part of an Applied Statistics course. The main objective is to explore and analyze the relationship between hardware characteristics and software calculation performance through multivariate statistical techniques.

> **Note**: This project acts as a clean, standardized template for Applied Statistics analyses using modern R practices.

---

## 🎯 Project Overview

The project aims to evaluate how different hardware metrics and system conditions impact the calculation performance of a given software. The target variable is **`y_prestazSWcalc`** (Software Calculation Performance), while the predictor variables include:
- `x1_CPU`: CPU Usage (%)
- `x2_HD`: Hard Drive Storage (GB)
- `x3_proc`: Number of Running Processes
- `x4_aging`: PC Age (Years)
- `x5_audio`: Audio playing status (Binary)
- `x6_RAM`: RAM Usage (GB)

Key methodological steps include:
- **Exploratory Data Analysis (EDA)**: Visualizing distributions using histograms and boxplots.
- **Bivariate Analysis**: Investigating relationships between the target variable and predictors through scatter plots and polynomial regression curves.
- **Correlation Analysis**: Generating correlation matrices to assess multicollinearity.
- **Multiple Linear Regression**: Building and evaluating an initial full linear model.
- **Feature Selection**: Applying Stepwise Backward Elimination to optimize the model.
- **Residual Diagnostics**: Validating the assumptions of the final optimized model.

## 📦 Repository Structure

The project is structured to ensure reproducibility and ease of use:

- **`dataset/`**: Folder containing the original raw datasets (`.csv`).
- **`docs/`**:
  - `progetto_descrizione.pdf`: Project guidelines and specifications.
  - `relazione_finale_gruppo_9.pdf`: The final, comprehensive technical report detailing models, interpretations, and results.
- **`statistical_analysis.R`**: The core R script containing the entire analytical pipeline (data loading, EDA, modeling, and visualization).
- **`generate_mock_dataset.R`**: A utility script to generate a synthetic dataset (`Dataset_AH_gruppo9.csv`). This ensures the project can be run out-of-the-box even if the original dataset is unavailable.
- **`install_requirements.R`**: A setup script that automatically installs and loads all required R packages into a local library structure (`.Rlib`).
- **`presentazione_gruppo_9.pptx`**: Presentation slides summarizing the methodology and key findings.
- **`Rplots.pdf`**: Autogenerated visual output from the R scripts.

##  Getting Started

Follow these steps to replicate the analysis locally:

1. **Clone the repository**:
   ```bash
   git clone https://github.com/your-username/your-repo-name.git
   cd your-repo-name
   ```
2. **Open the project** in RStudio.
3. **Install Dependencies**: 
   Source or run the `install_requirements.R` script. This handles all package installations cleanly into a localized `.Rlib` folder, avoiding system-wide clutter.
4. **Prepare the Data**:
   The script expects a file named `Dataset_AH_gruppo9.csv` at the root directory. If you don't have the original dataset, simply run:
   ```R
   source("generate_mock_dataset.R")
   ```
   This will create a realistic dummy dataset.
5. **Run the Analysis**: 
   Open and execute `statistical_analysis.R`. The console will print model summaries and ANOVA comparisons, while graphical panels (EDA, Models, Residuals) will be generated.

##  Key Technical Highlights

- **Fully Reproducible Pipeline**: From dependency installation to dataset mocking, the analysis is designed to work smoothly on any machine.
